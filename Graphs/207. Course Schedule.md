#  207. Course Schedule

## Problem Statement

There are a total of `numCourses` courses you have to take, labeled from `0` to `numCourses-1`. You are given an array `prerequisites` where `prerequisites[i] = [a_i, b_i]` indicates that you must take course `b_i` before taking course `a_i`.

Return `true` if you can finish all courses. Otherwise, return `false`.

### Example 1:
```
Input: numCourses = 2, prerequisites = [[1,0]]
Output: true
Explanation: There are a total of 2 courses to take. To take course 1 you should have finished course 0. So it is possible.
``` 

### Example 2:
```
Input: numCourses = 2, prerequisites = [[1,0],[0,1]]
Output: false
Explanation: There are a total of 2 courses to take. To take course 1 you should have finished course 0, and to take course 0 you should also have finished course 1. So it is impossible.
``` 

---

## Approach

This problem can be solved using `Topological Sort` or `Kahn's Algorithm`. What does it imply? It means that we can only take a course if all its prerequisites have been taken. 

We can represent the courses and their prerequisites as a directed graph, where each course is a node and there is a directed edge from course `b_i` to course `a_i` if `b_i` is a prerequisite for `a_i`.

For performing `kahn's algorithm`, we need to keep track of the indegree of each node (course). The indegree of a node is the number of edges coming into it, which in this case represents the number of prerequisites for that course.

Declare a queue and enqueue all courses that have an indegree of 0 (i.e., courses that have no prerequisites). 

Then, we can repeatedly dequeue a course from the queue, mark it as completed, and reduce the indegree of its neighboring courses (i.e., courses that depend on it). If any neighboring course's indegree becomes 0, we can enqueue it as well.

If we can complete all courses (i.e., if the number of completed courses equals `numCourses`), then we can return `true`. Otherwise, we return `false`.

---

## Code Implementation

```cpp
class Solution {
public:
    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
        vector<vector<int>> adjList(numCourses);
        vector<int> indegrees(numCourses);
        queue<int> q;

        for(auto &prereq: prerequisites){
            int u = prereq[0], v = prereq[1];
            adjList[u].push_back(v);
            indegrees[v]++;
        }

        for(int i = 0; i < numCourses; i++){
            if(indegrees[i] == 0) q.push(i);
        }
        int completedCourses = 0;
        
        while(!q.empty()){
            int node = q.front(); q.pop();
            completedCourses++;

            for(auto &neighbor: adjList[node]){
                indegrees[neighbor]--;
                if(indegrees[neighbor] == 0){
                    q.push(neighbor);
                }
            }
        }
        return numCourses == completedCourses; 
    }
};
```

---

## Complexity Analysis

- **Time Complexity**: O(V + E), where V is the number of courses and E is the number of prerequisites. We need to traverse all courses and prerequisites to build the adjacency list and indegree array, and then we may need to visit each course once in the worst case.

- **Space Complexity**: O(V + E) for the adjacency list and indegree array. The queue can hold at most V courses in the worst case, so it also contributes O(V) space.
