# 25. Reverse Nodes in k-Group

## Problem Statement

Given the head of a linked list, reverse the nodes of the list `k` at a time, and return the modified list.

### Example 1:

```
Input: head = [1,2,3,4,5], k = 2
Output: [2,1,4,3,5]
```

### Example 2:

```
Input: head = [1,2,3,4,5], k = 3
Output: [3,2,1,4,5]
```

---

## Approach

As the problem title states, we have to `reverse` the nodes of a linked list `k` at a time.

To reverse the nodes of a linked list, we can use three pointers: `prev`, `curr`, and `front`.

- `prev` will point to the previous node of the current node.
- `curr` will point to the current node that we are processing.
- `front` will point to the next node of the current node.

We need two helper functions to solve this problem:

1. `findKthNode(ListNode head, int k)`: This function will find the `k`-th node from the current node. If there are less than `k` nodes remaining, it will return `null`.

2. `reverseNodes(ListNode head)`: This function will reverse the nodes of the linked list starting from the given head node and return the new head of the reversed linked list.

We traverse the linked list in groups of `k` nodes. For each group of `k` nodes, we reverse the nodes and connect the reversed group with the previous and next groups.

Certain edge cases to consider:

1. If the number of nodes in the linked list is not a multiple of `k`, then the last remaining nodes should remain as they are.

2. If `k` is 1, then the linked list should remain unchanged.

3. If `k` is greater than the number of nodes in the linked list, then the linked list should remain unchanged.

4. If the linked list is empty, then we should return `null`.

In this approach, we will be reversing the linked list in place, which means we will not be using any additional data structures to store the nodes. We will be modifying the pointers of the existing nodes to achieve the desired result.


---

## Code Implementation

```java
class Solution {
    private ListNode findKthNode(ListNode head, int k){
        ListNode temp = head;
        int curr = 1;
        while(temp != null){
            if(curr == k) return temp;
            temp = temp.next;
            curr++;
        }
        return temp;
    }

    private ListNode reverseNodes(ListNode head){
        if(head == null || head.next == null) return head;
        ListNode front = null, prev = null, temp = head;
        while(temp != null){
            front = temp.next;
            temp.next = prev;
            prev = temp;
            temp = front;
        }
        return prev;
    }

    public ListNode reverseKGroup(ListNode head, int k) {
        ListNode currNode = head;
        ListNode prevNode = null, nextNode = null;

        while(currNode != null){
            ListNode kthNode = findKthNode(currNode, k);
            if(kthNode == null){
                prevNode.next = currNode;
                break;
            }

            nextNode = kthNode.next;
            kthNode.next = null;
            reverseNodes(currNode);

            if(currNode == head){
                head = kthNode;
            }
            else{
                prevNode.next = kthNode;
            }
            prevNode = currNode;
            currNode = nextNode;
        }
        return head;
    }   
}
```
---

## Complexity Analysis

- **Time Complexity**: `O(n)`, where `n` is the number of nodes in the linked list. We traverse the linked list once to reverse the nodes in groups of `k`.

- **Space Complexity**: `O(1)`, as we are reversing the linked list in place without using any additional data structures.
