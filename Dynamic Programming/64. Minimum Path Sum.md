# 64. Minimum Path Sum

## Problem Statement

Given a `m x n` grid filled with non-negative numbers, find a path from top left to bottom right which minimizes the sum of all numbers along its path.

### Example 1:
```
Input: grid = [[1,3,1],[1,5,1],[4,2,1]]
Output: 7
Explanation: Because the path 1 → 3 → 1 → 1 → 1 minimizes the sum.
```

### Example 2:
```
Input: grid = [[1,2,3],[4,5,6]]
Output: 12
```

---

## Approach

What can we observe about the path from the top-left corner to the bottom-right corner?

- The path can only move right or down at any point in time.

- The minimum path sum to reach a cell `(i, j)` can be expressed as the minimum of the path sums to reach the cell above it `(i - 1, j)` and the cell to the left of it `(i, j - 1)`, plus the value of the current cell `grid[i][j]`.

This leads us to a dynamic programming approach where we can create a 2D DP table `dp` of the same size as the input grid, where `dp[i][j]` represents the minimum path sum to reach cell `(i, j)`.

So we just need to check at `[n - 1][m - 1]` to get the minimum path sum from the top-left corner to the bottom-right corner.

---

## Code Implementation

```cpp
class Solution {
public:
    int n, m;
    vector<vector<int>> dp;
    int minPath(int i, int j, vector<vector<int>> &grid){
        if(i == n - 1 && j == m - 1) return grid[i][j];
        if(dp[i][j] != -1) return dp[i][j];
        
        int down = INT_MAX; int right = INT_MAX;
        if(i + 1 < n){
            down = grid[i][j] + minPath(i + 1, j, grid);
        }
        if(j + 1 < m){
            right = grid[i][j] + minPath(i, j + 1, grid);
        }
        return dp[i][j] = min(down, right);
    }
    
    int minPathSum(vector<vector<int>>& grid) {
        this->n = grid.size(), this->m = grid[0].size();
        this->dp.assign(n, vector<int> (m, -1));
        return minPath(0, 0, grid);
    }
};
```

---

## Complexity Analysis

- **Time Complexity**: O(n * m), where n is the number of rows and m is the number of columns in the grid. This is because we are filling a DP table of size n x m.

- **Space Complexity**: O(n * m) for the DP table. However, we can optimize this to O(m) by only keeping track of the current and previous rows, since we only need the values from the previous row to calculate the current row.
