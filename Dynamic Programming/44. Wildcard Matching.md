# 44. Wildcard Matching

## Problem Statement

Given an input string `s` and a pattern `p`, implement wildcard pattern matching with support for `'?'` and `'*'` where:

- `'?'` Matches any single character.
- `'*'` Matches any sequence of characters (including the empty sequence).

The matching should cover the entire input string (not partial).

### Example 1:

```
Input: s = "aa", p = "a"
Output: false
Explanation: "a" does not match the entire string "aa".
```

### Example 2:

```
Input: s = "babcdfne", p = "b?bc?f*"
Output: true
Explanation: The pattern "b?bc?f*" matches the string "babcdfne" because:
- 'b' matches 'b'
- '?' matches 'a'
- 'b' matches 'b'
- 'c' matches 'c'
- '?' matches 'd'
- 'f' matches 'f'
- '*' matches 'ne'
```

---

## Approach

Understand when two characters match:
- If the characters are the same, they match.
- If the pattern character is '?', it matches any single character in the string.
- If the pattern character is '*', it can match zero or more characters in the string.

With this understanding, we can use a recursive approach with memoization to avoid redundant calculations. We will define a recursive function `matchChars(i, j)` that checks if the substring of `s` up to index `i` matches the substring of `p` up to index `j`.

1. If both `i` and `j` are less than 0, it means we have successfully matched the entire string and pattern, so we return true.

2. If `j` is less than 0 but `i` is greater than or equal to 0, it means the pattern is exhausted but the string is not, so we return false.

3. If `i` is less than 0 but `j` is greater than or equal to 0, we need to check if the remaining characters in the pattern are all '*'. If they are, we can match the empty sequence, so we return true. Otherwise, we return false.

4. If the result for `matchChars(i, j)` is already computed and stored in the `dp` table, we return that result to avoid redundant calculations.

5. If the characters at `s[i]` and `p[j]` match or if `p[j]` is '?', we can move both indices back by one and check for a match.

6. If `p[j]` is '*', we have two options:
   - We can treat '*' as matching zero characters, so we move the pattern index back by one and check for a match.
   - We can treat '*' as matching one or more characters, so we move the string index back by one and check for a match.

7. If none of the above conditions are met, it means the characters do not match, and we return false.

---

## Code Implementation

```java
class Solution {
    int[][] dp;
    private boolean matchChars(int i, int j, String s, String p){
        if(i < 0 && j < 0) return true;
        if(j < 0 && i >= 0) return false;
        if(i < 0 && j >= 0){
            for(int idx = j; idx >= 0; idx--){
                if(p.charAt(idx) != '*') return false;
            }
            return true;
        }
        if(dp[i][j] != -1) return dp[i][j] == 1;

        if(s.charAt(i) == p.charAt(j) || p.charAt(j) == '?'){
            boolean res = matchChars(i - 1, j - 1, s, p);
            dp[i][j] = res ? 1 : 0;
            return res;
        }

        if(p.charAt(j) == '*'){
            boolean res = matchChars(i - 1, j, s, p) || matchChars(i, j - 1, s, p);
            dp[i][j] = res ? 1 : 0;
            return res;
        }
        
        dp[i][j] = 0;
        return false;
    }
    
    public boolean isMatch(String s, String p) {
        int n = s.length(), m = p.length();
        this.dp = new int[n + 1][m + 1];
        for(int i = 0; i <= n; i++) Arrays.fill(dp[i], -1);
        return matchChars(n - 1, m - 1, s, p);
    }
}
```

---

## Complexity Analysis

- **Time Complexity**: O(n * m), where n is the length of the input string `s` and m is the length of the pattern `p`. This is because we are using memoization to store results for each combination of indices in `s` and `p`.

- **Space Complexity**: O(n * m), where n is the length of the input string `s` and m is the length of the pattern `p`. This is due to the memoization table `dp` that we use to store results for each combination of indices in `s` and `p`.
