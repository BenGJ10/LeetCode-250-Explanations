# 84. Largest Rectangle in Histogram

## Problem Statement

Given an array of integers representing the height of bars in a histogram, find the area of the largest rectangle that can be formed within the histogram.

### Example 1:

```
Input: heights = [2,1,5,6,2,3]
Output: 10
Explanation: The largest rectangle can be formed using the bars with heights 5 and 6, which gives an area of 5 * 2 = 10.
```

### Example 2:

```
Input: heights = [2,4]
Output: 4
Explanation: The largest rectangle can be formed using the bar with height 4, which gives an area of 4 * 1 = 4.
```

---

## Approach

We have to think in terms of the largest rectangle that can be formed using each bar as the height of the rectangle. For each bar, we need to find the width of the rectangle that can be formed with that bar as the height.

We can use the `monotonic stack` approach to find the next smaller element to the left and right of each bar. The width of the rectangle for each bar can be calculated as the distance between the next smaller elements on both sides.

1. Initialize a stack to keep track of the indices of the bars.

2. Iterate through the heights array.

3. For each bar, while the stack is `not empty` and the current bar's height is `less than or equal to` the height of the bar at the index stored at the top of the stack, pop the stack. This means we have found the next smaller element for the bar at the popped index.

4. Calculate the `area` of the rectangle that can be formed with the popped bar as the height. The width can be calculated as the distance between the current index and the index of the next smaller element on the left.

5. Update the `maxArea` if the calculated area is greater than the current `maxArea`.

6. After processing all bars, there may still be bars left in the stack. For each remaining bar, calculate the area using the same logic as above, considering the right boundary as the end of the histogram.

7. Return the `maxArea` as the result.

---

## Code Implementation

```java
class Solution {
    public int largestRectangleArea(int[] heights) {
        int n = heights.length;
        int maxArea = 0;
        Stack<Integer> st = new Stack<>();
        
        for(int i = 0; i < n; i++){
            while(!st.isEmpty() && heights[st.peek()] >= heights[i]){
                int midElem = heights[st.peek()]; st.pop();
                int prevElem = st.isEmpty() ? -1 : st.peek();
                int nextElem = i;
                int currArea = (nextElem - prevElem - 1) * midElem;
                maxArea = Math.max(maxArea, currArea);
            }            
            st.push(i);
        }

        while(!st.isEmpty()){
            int midElem = heights
            
            [st.peek()]; st.pop();
            int prevElem = st.isEmpty() ? -1 : st.peek();
            int nextElem = n;
            int currArea = (nextElem - prevElem - 1) * midElem;
            maxArea = Math.max(maxArea, currArea);
        }

        return maxArea;
    }
}
```

--- 

## Complexity Analysis

- **Time Complexity**: O(n), where n is the number of bars in the histogram. Each bar is pushed and popped from the stack at most once.

- **Space Complexity**: O(n), where n is the number of bars in the histogram. We use a stack to keep track of the indices of the bars.
