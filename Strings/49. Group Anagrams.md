# 49. Group Anagrams

## Problem Statement

Given an array of strings `strs`, group the anagrams together. You can return the answer in any order.

An `anagram` is a word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once.

### Example 1:
```
Input: strs = ["eat","tea","tan","ate","nat","bat"]
Output: [["bat"],["nat","tan"],["ate","eat","tea"]]
```

### Example 2:
```
Input: strs = [""]
Output: [[""]]
```

### Example 3:
```
Input: strs = ["a"]
Output: [["a"]]
```

---

## Approach

To group anagrams together, we can use a `hash map` (or `dictionary`) to store the anagrams. The key in the hash map will be a string that represents the frequency of characters in the anagram, and the value will be a list of strings that are anagrams of each other.

To create the key for the hash map, we can count the frequency of each character in the string and create a string representation of this frequency. For example, if the string is "eat", the frequency of characters would be `{'e': 1, 'a': 1, 't': 1}`, and we can represent this as "a1e1t1". This way, all anagrams will have the same key.

1. Initialize an empty hash map to store the anagrams.

2. Iterate through each string in the input array:
   - For each string, count the `frequency` of each character and create a key based on this frequency.

   - If the `key` does not exist in the hash map, create a new entry with an empty list.

    - Append the original string to the list corresponding to the `key` in the hash map.

3. Finally, return the values of the hash map as a list of lists, which will contain the grouped anagrams.

---

## Code Implementation

```java
class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        int n = strs.length;
        Map<String, List<String>> map = new HashMap<>();

        for(int i = 0; i < n; i++){
            String str = strs[i];
            
            int[] freq = new int[26];
            for(int j = 0; j < str.length(); j++){
                freq[str.charAt(j) - 'a']++;
            }

            StringBuilder key = new StringBuilder();
            for(int j = 0; j < 26; j++){
                key.append(String.valueOf((char)(j + 'a')).repeat(freq[j]));
            }

            map.putIfAbsent(key.toString(), new ArrayList<>());
            map.get(key.toString()).add(str);
        }

        List<List<String>> result = new ArrayList<>();
        for(List<String> values: map.values()){
            result.add(values);
        }
        return result;
    }
}
```

---

## Complexity Analysis

- **Time Complexity**: O(N * K), where N is the number of strings in the input array and K is the maximum length of a string in the input array. This is because we iterate through each string and calculate its frequency, which takes O(K) time.

- **Space Complexity**: O(N * K), where N is the number of strings in the input array and K is the maximum length of a string in the input array. This is because in the worst case, all strings could be anagrams of each other, and we would store all of them in the map.
