# 496. Next Greater Element I

## Problem Statement

You are given two arrays (without duplicates) `nums1` and `nums2` where `nums1`’s elements are a subset of `nums2`. Find all the next greater numbers for `nums1`'s elements in the corresponding places of `nums2`.

The Next Greater Number of a number `x` in `nums1` is the first greater number to its right in `nums2`. If it does not exist, output -1 for this number.

### Example 1:

```
Input: nums1 = [4,1,2], nums2 = [1,3,4,2]
Output: [-1,3,-1]
Explanation:
For number 4 in nums1, there is no greater number to its right in nums2.
For number 1 in nums1, the next greater number to its right in nums2 is 3.
For number 2 in nums1, there is no greater number to its right in nums2.
``` 

### Example 2:

```
Input: nums1 = [2,4], nums2 = [1,2,3,4]
Output: [3,-1]
Explanation:
For number 2 in nums1, the next greater number to its right in nums2 is 3.
For number 4 in nums1, there is no greater number to its right in nums2.
```

---

## Approach

This is a `monotomic stack` problem. We can use a stack to keep track of the elements in `nums2` and find the next greater element for each number.

We can use a hashmap to store the next greater element for each number in `nums2`. Then, we can iterate through `nums1` and get the next greater element from the hashmap.

1. Initialize a stack and a hashmap.

2. Iterate through `nums2`:
   - While the stack is not empty and the current number is greater than the number at the top of the stack, pop the stack and store the current number as the next greater element for the popped number in the hashmap.
   - Push the current number onto the stack.

3. After processing `nums2`, iterate through `nums1` and for each number, check if it exists in the hashmap. If it does, add the corresponding next greater element to the result array; otherwise, add -1.

4. Return the result array.

---

## Code Implementation

```java
class Solution {
    public int[] nextGreaterElement(int[] nums1, int[] nums2) {
        int n = nums1.length, m = nums2.length;
        int[] res = new int[n];
        Arrays.fill(res, -1);

        Map<Integer, Integer> map = new HashMap<>();
        Stack<Integer> st = new Stack<>();

        for(int i = 0; i < m; i++){
            while(!st.isEmpty() && nums2[i] >= st.peek()){
                map.put(st.peek(), nums2[i]);
                st.pop();
            }
            st.push(nums2[i]);
        }

        for(int i = 0; i < n; i++){
            int key = nums1[i];
            if(map.containsKey(key)){
                res[i] = map.get(key);
            }
        }
        return res;
    }
}
```

---

## Complexity Analysis

- **Time Complexity:** O(n + m), where n is the length of `nums1` and m is the length of `nums2`. We traverse both arrays once.

- **Space Complexity:** O(m), where m is the length of `nums2`. We use a stack and a hashmap to store the next greater elements.
