# 90. Subsets II

## Problem Statement

Given an integer array `nums` that may contain duplicates, return all possible subsets (the power set).

A solution set must not contain duplicate subsets. Return the solution in any order.

### Example 1:
```
Input: nums = [1,2,2]
Output: [[],[1],[1,2],[1,2,2],[2],[2,2]]
```

### Example 2:
```
Input: nums = [0]
Output: [[],[0]]    
```

---

## Approach

We are tasked to find all possible subsets of a given set of integers, including duplicates.

1. **Sorting the Input**: First, we sort the input array `nums`. This helps in easily identifying duplicates and ensuring that we do not generate duplicate subsets.

2. **Backtracking**: We use a backtracking approach to generate all possible subsets. We maintain a current subset and explore two choices at each step: include the current number or skip it.

3. Initially, we start with an empty subset and iterate through the sorted array. For each number, we can either include it in the current subset or skip it. If we include it, we move to the next index. 

4. If we skip the current number, we need to ensure that we do not include the same number again in the next recursive call. This is done by skipping all subsequent duplicates of the current number.

5. **Base Case**: When we reach the end of the array, we add the current subset to the result list.

---

## Code Implementation

```java
class Solution {
    List<List<Integer>> res;
    
    private void backtrack(int index, int[] nums, List<Integer> curr){
        if(index >= nums.length){
            res.add(new ArrayList<>(curr));
            return;
        }

        curr.add(nums[index]);
        backtrack(index + 1, nums, curr);
        curr.remove(curr.size() - 1);
        while(index + 1 < nums.length && nums[index] == nums[index + 1]){
            index++;
        }
        backtrack(index + 1, nums, curr);
    }

    public List<List<Integer>> subsetsWithDup(int[] nums) {
        Arrays.sort(nums);
        this.res = new ArrayList<>();
        List<Integer> curr = new ArrayList<>();
        backtrack(0, nums, curr);
        return this.res;
    }
}
```

---

## Complexity Analysis

- **Time Complexity**: O(2^n), where `n` is the length of the input array `nums`. This is because we are generating all possible subsets, and there are 2^n subsets for a set of size n.

- **Space Complexity**: O(n), where `n` is the length of the input array `nums`. This is due to the recursion stack and the space used to store the current subset.
