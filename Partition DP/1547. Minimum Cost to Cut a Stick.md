# 1547. Minimum Cost to Cut a Stick

We have a wooden stick of length `n` units. The stick is labelled from `0` to `n`. For example, a stick of length `6` is labelled as follows:

```
0 1 2 3 4 5 6
```

We are given an integer array `cuts` where `cuts[i]` denotes a position you should perform a cut at.

The cost of each cut is equal to the length of the stick being cut. The goal is to find the minimum total cost to make all the cuts. When you cut a stick, it will be split into two smaller sticks (i.e. the sum of their lengths is the length of the stick before the cut). Please refer to the first example for a better explanation.

Return the `minimum` total cost of the cuts.

### Example 1:

```
Input: n = 7, cuts = [1,3,4,5]
Output: 16
Explanation: The optimal way to cut the stick is as follows:
1. Cut the stick at position 3. The cost is 7. Now we have two sticks: [0,3] and [3,7].
2. Cut the stick [0,3] at position 1. The cost is 3. Now we have two sticks: [0,1] and [1,3].
3. Cut the stick [3,7] at position 5. The cost is 4. Now we have two sticks: [3,5] and [5,7].
4. Cut the stick [3,5] at position 4. The cost is 2. Now we have two sticks: [3,4] and [4,5].
The total cost is 7 + 3 + 4 + 2 = 16.
```

---

## Approach

This problem is a classic example of **Partition DP**.

The key observation is that **every possible cut can be chosen as the first cut**. Once we make that cut, the stick is divided into two **independent** subproblems:

* Cut the left part.
* Cut the right part.

The total cost for choosing a cut `c` is:

```text
Current Stick Cost
+ Cost of Left Partition
+ Cost of Right Partition
```

This gives us the recurrence:

```text
dp(i, j) =
min(
    dp(i, c - 1)
  + dp(c + 1, j)
  + (cuts[j + 1] - cuts[i - 1])
)
```

where:

* `i` and `j` represent the range of **cut indices** that are still to be performed.
* `c` is the cut chosen as the **first cut** within the current range.
* `cuts[j + 1] - cuts[i - 1]` gives the length of the current stick, which is the cost of making the first cut.

### Steps

1. Sort the `cuts` array and add the two boundaries (`0` and `n`).
2. Define `dp(i, j)` as the minimum cost to perform all cuts from index `i` to `j`.
3. Try every cut `c` in the range `[i, j]` as the first cut.
4. Recursively compute the cost of the left and right partitions and add the current stick length.
5. Memoize each `(i, j)` state to avoid recomputation.

### Partition DP Pattern

Most Partition DP problems follow the same template:

```text
dp(i, j) = best over all partitions k

{
    dp(i, k - 1)
  + dp(k + 1, j)
  + cost(i, k, j)
}
```

The only thing that changes from one problem to another is the definition of `cost(i, k, j)`. In this problem, it is simply the **length of the current stick**.

## Code Implementation

```java
class Solution {
    int[][] dp;
    
    private int partitionDP(int i, int j, List<Integer> cuts){
        if(i > j) return 0;
        if(dp[i][j] != -1) return dp[i][j];
        
        int minCostReq = Integer.MAX_VALUE;
        for(int c = i; c <= j; c++){
            int leftHalf = partitionDP(i, c - 1, cuts);
            int rightHalf = partitionDP(c + 1, j, cuts);
            int currCostReq = leftHalf + rightHalf + (cuts.get(j + 1) - cuts.get(i - 1));
            minCostReq = Math.min(minCostReq, currCostReq);
        }
        
        return dp[i][j] = minCostReq;
    }

    public int minCost(int n, int[] inputs) {
        int m = inputs.length;
        this.dp = new int[m + 2][m + 2];
        for(int i = 0; i < m + 2; i++) Arrays.fill(dp[i], -1);
       
        List<Integer> cuts = new ArrayList<>();        
        cuts.add(0);
        for(int i = 0; i < m; i++){
            cuts.add(inputs[i]);
        }
        cuts.add(n);
        Collections.sort(cuts);

        return partitionDP(1, m, cuts);
    }
}
```

---

## Complexity Analysis

- **Time Complexity:** O(m^3), where m is the number of cuts. The recursive function explores all possible partitions of the cuts, and for each partition, it calculates the cost of cutting the stick.

- **Space Complexity:** O(m^2), where m is the number of cuts. The space is used for the memoization table `dp` to store the results of subproblems.
