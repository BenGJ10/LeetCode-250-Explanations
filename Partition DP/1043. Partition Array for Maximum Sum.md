# 1043. Partition Array for Maximum Sum

## Problem Statement

Given an integer array `arr`, partition the array into contiguous subarrays of length at most `k`. After partitioning, each subarray's elements are changed to the maximum value in that subarray. Return the largest possible sum of the modified array.

### Example 1:

```
Input: arr = [1,15,7,9,2,5,10], k = 3
Output: 84
Explanation: arr becomes [15,15,15,9,10,10,10]
```

### Example 2:

```
Input: arr = [1,4,1,5,7,3,6,1,9,2,8], k = 4
Output: 83
```

--- 

## Approach

We can solve this problem using a recursive approach with memoization (top-down dynamic programming). The idea is to iterate through the array and for each index, we can try to partition the array into subarrays of length at most `k`. 

For each partition, we calculate the maximum element in that subarray and compute the sum of the modified array. We then recursively call the function for the next index after the current partition and keep track of the maximum sum we can achieve.

1. Create a recursive function `partition(idx)` that takes the current index as an argument.

2. If the current index is equal to the length of the array, return 0 (base case).

3. If the result for the current index is already computed (i.e., `dp[idx] != -1`), return the stored result.

4. Initialize variables to keep track of the length of the current partition, the maximum element in the current partition, and the maximum sum we can achieve.

5. Iterate through the array starting from the current index up to `idx + k` (or the end of the array). For each index `i`, update the length of the current partition and the maximum element in that partition.

6. Calculate the sum of the current partition by multiplying the maximum element with the length of the partition and adding the result of the recursive call for the next index (`partition(i + 1)`).

7. Update the maximum sum if the current sum is greater than the previously recorded maximum sum.

8. Store the computed maximum sum in the `dp` array for the current index and return it.

---

## Code Implementation

```java
class Solution {
    int dp[];
    
    private int partition(int idx, int[] arr, int k){
        if(idx == arr.length) return 0;
        if(dp[idx] != -1) return dp[idx];

        int len = 0;
        int maxSum = Integer.MIN_VALUE, maxElem = Integer.MIN_VALUE;
        
        for(int i = idx; i < Math.min(arr.length, idx + k); i++){
            len++;
            maxElem = Math.max(maxElem, arr[i]);
            int nextHalf = partition(i + 1, arr, k);
            int currSum = (maxElem * len) + nextHalf;
            maxSum = Math.max(maxSum, currSum);
        }
        return dp[idx] = maxSum;
    }
    
    public int maxSumAfterPartitioning(int[] arr, int k) {
        int n = arr.length;
        this.dp = new int[n];
        Arrays.fill(dp, -1);
        return partition(0, arr, k);
    }
}
```

---

## Complexity Analysis

- **Time Complexity**: O(n * k), where n is the length of the array. For each index, we may look ahead up to k elements.

- **Space Complexity**: O(n) for the DP array and O(n) for the recursion stack in the worst case, leading to a total of O(n) space complexity.
