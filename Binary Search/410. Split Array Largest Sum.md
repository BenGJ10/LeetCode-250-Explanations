# 410. Split Array Largest Sum

## Problem Statement

Given an array `nums` which consists of non-negative integers and an integer `m`, you can split the array into `m` non-empty continuous subarrays. Write an algorithm to minimize the largest sum among these `m` subarrays.

### Example 1

```
Input: nums = [7,2,5,10,8], m = 2
Output: 18
Explanation: There are four ways to split nums into two subarrays.
The best way is to split it into [7,2,5] and [10,8], where the largest sum among the two subarrays is only 18.
``` 

### Example 2

```
Input: nums = [1,2,3,4,5], m = 2
Output: 9
Explanation: There are four ways to split nums into two subarrays.
The best way is to split it into [1,2,3] and [4,5], where the largest sum among the two subarrays is only 9.
```

---

## Approach

When we check the test cases, we can see that the answer will be between the `maximum` element in the array and the `sum` of all elements in the array. We can use `binary search` to find the minimum largest sum among the `m` subarrays.

1. We will initialize `low` to the maximum element in the array and `high` to the sum of all elements in the array.

2. We will perform binary search between `low` and `high`. For each `mid` value, we will check if it is possible to split the array into `m` subarrays such that the largest sum among these subarrays is less than or equal to `mid`.

3. If it is possible to split the array with the current `mid`, we will update our result and move the `high` pointer to `mid - 1` to find a smaller largest sum.

4. If it is not possible to split the array with the current `mid`, we will move the `low` pointer to `mid + 1` to find a larger largest sum.


---

## Code Implementation

```java
class Solution {
    int n;
    private boolean canPartition(int[] nums, int k, int sum){
        int parts = 1;
        int currSum = 0;
        for(int i = 0; i < n; i++){
            if(currSum + nums[i] <= sum){
                currSum += nums[i];
            }
            else{
                parts++;
                currSum = nums[i];
            }
        }
        return parts <= k;
    }

    public int splitArray(int[] nums, int k) {
        this.n = nums.length;
        int low = Integer.MIN_VALUE, high = 0, res = 0;
        for(int i = 0; i < n; i++){
            low = Math.max(low, nums[i]);
            high += nums[i];
        }

        while(low <= high){
            int mid = low + (high - low) / 2;
            if(canPartition(nums, k, mid)){
                res = mid;
                high = mid - 1;
            }
            else{
                low = mid + 1;
            }
        }
        return res;
    }
}
```

---

## Complexity Analysis

- **Time Complexity**: O(n log m), where n is the length of the input array and m is the sum of all elements in the array. The binary search runs in O(log m) and the `canPartition` function runs in O(n).

- **Space Complexity**: O(1), as we are using only a constant amount of extra space for variables.
