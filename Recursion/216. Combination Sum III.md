# 216. Combination Sum III

## Problem Statement

Find all valid combinations of `k` numbers that sum up to `n` such that the following conditions are true:

- Only numbers 1 through 9 are used.
- Each number is used at most once.


### Example 1:

```
Input: k = 3, n = 7
Output: [[1,2,4]]
Explanation:
1 + 2 + 4 = 7
There are no other valid combinations.
```

### Example 2:

```
Input: k = 3, n = 9
Output: [[1,2,6], [1,3,5], [2,3,4]]
Explanation:
1 + 2 + 6 = 9
1 + 3 + 5 = 9
2 + 3 + 4 = 9
There are no other valid combinations.
```

---

## Approach

We are given `k` and `n`, and we need to find all combinations of `k` numbers that sum up to `n`. The numbers can only be from 1 to 9, and each number can be used at most once.

1. We can use a backtracking approach to explore all possible combinations of numbers from 1 to 9.

2. We will maintain a list to store the current combination and a result list to store all valid combinations.

3. We will start from the first number (1) and try to include it in the current combination. If the current combination has `k` numbers and sums up to `n`, we will add it to the result list.

4. If the current combination has more than `k` numbers or the sum exceeds `n`, we will backtrack and try the next number.

5. When we reach the end of the numbers (9), we will return and explore other combinations.

6. Finally, we will return the result list containing all valid combinations.

---

## Code Implementation

```java
class Solution {
    List<List<Integer>> res;
    
    private void backtrack(int idx, int k, int n, List<Integer> curr){
        if(k == 0){
            if(n == 0){
                res.add(new ArrayList<>(curr));
            }
            return;
        }

        for(int i = idx; i <= 9; i++){
            if(n < 0) break;                        
            curr.add(i);
            backtrack(i + 1, k - 1, n - i, curr);
            curr.remove(curr.size() - 1);
        }
    }

    public List<List<Integer>> combinationSum3(int k, int n) {
        this.res = new ArrayList<>();
        List<Integer> curr = new ArrayList<>();        
        backtrack(1, k, n, curr);
        return this.res;
    }
}
```

---

## Complexity Analysis

- **Time Complexity**: O(2^n) - In the worst case, we may have to explore all possible combinations of numbers from 1 to 9, which can lead to an exponential number of combinations.

- **Space Complexity**: O(k) - The space used by the recursion stack and the current combination list can go up to `k`, which is the number of elements in each combination.
