# 743. Network Delay Time

## Problem Statement

You are given a network of `n` nodes, labeled from `1` to `n`. You are also given `times`, a list of travel times as directed edges `times[i] = (ui, vi, wi)`, where `ui` is the source node, `vi` is the target node, and `wi` is the time it takes for a signal to travel from source to target.

We will send a signal from a given node `k`. Return the time it takes for all the `n` nodes to receive the signal. If it is impossible for all the `n` nodes to receive the signal, return `-1`.

### Example 1:

```
Input: times = [[2,1,1],[2,3,1],[3,4,1]], n = 4, k = 2
Output: 2
```

### Example 2:

```
Input: times = [[1,2,1]], n = 2, k = 1
Output: 1
```

### Example 3:

```
Input: times = [[1,2,1]], n = 2, k = 2
Output: -1
```
---

## Approach

This is a classic shortest path problem in a directed graph. We can use `Dijkstra's algorithm` to find the shortest time it takes for the signal to reach all nodes starting from node `k`.

1. Create an adjacency list to represent the graph from the given `times` input.

2. Use a priority queue (min-heap) to keep track of the next node to process based on the shortest time found so far.

3. Initialize a distance array to keep track of the shortest time to reach each node, and set the distance to the starting node `k` to `0`.

4. While the priority queue is not empty, pop the node with the smallest time and update the distances to its neighbors. If a shorter time is found for a neighbor, update the distance and push it into the priority queue.

5. After processing all reachable nodes, check the distance array. If any node is still at `INT_MAX`, it means it is unreachable, and we return `-1`. Otherwise, return the maximum time from the distance array, which represents the time it takes for all nodes to receive the signal.

---

## Code Implementation

```cpp
class Solution {
public:
    int networkDelayTime(vector<vector<int>>& times, int n, int k) {
        vector<vector<pair<int, int>>> adjList(n + 1);
        for(auto &time: times){
            int u = time[0];
            int v = time[1];          
            int w = time[2];
            adjList[u].push_back({v, w});
        }
        priority_queue<pair<int, int>, vector<pair<int, int>>, 
            greater<pair<int, int>>> pq;
        vector<int> dist(n + 1, INT_MAX);

        pq.push({0, k});
        dist[k] = 0;

        while(!pq.empty()){
            auto [currTime, currNode] = pq.top(); pq.pop();

            for(auto &neigh: adjList[currNode]){
                int neighNode = neigh.first;
                int neighTime = neigh.second;
                int newTime = neighTime + currTime;
                if(newTime < dist[neighNode]){
                    dist[neighNode] = newTime;
                    pq.push({newTime, neighNode});
                }
            }
        }

        int minTime = 0;
        for(int i = 1; i <= n; i++){
            if(dist[i] == INT_MAX) return -1;
            minTime = max(minTime, dist[i]);
        }
        return minTime;
    }
};
```

---

## Complexity Analysis

- **Time Complexity**: O(E log V), where E is the number of edges and V is the number of vertices in the graph. This is because we are using a priority queue to process each edge.

- **Space Complexity**: O(V + E), where V is the number of vertices and E is the number of edges. This is because we are storing the graph in an adjacency list and also maintaining a distance array.
