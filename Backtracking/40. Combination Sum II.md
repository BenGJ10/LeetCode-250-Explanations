# 40. Combination Sum II

## Problem Statement

Given a collection of candidate numbers (`candidates`) and a target number (`target`), find all unique combinations in `candidates` where the candidate numbers sum to `target`.

Each number in `candidates` may only be used once in the combination.

### Example 1:
```
Input: candidates = [10,1,2,7,6,1,5], target = 8
Output:
[
  [1,1,6],
  [1,2,5],
  [1,7],
  [2,6]
]
```

### Example 2:
```
Input: candidates = [2,5,2,1,2], target = 5
Output:
[
  [1,2,2],
  [5]
]
```

---

## Approach

Whenever the question is demanding us to find all the unique `combinations` of a given set of numbers, we can immediately think of using the `backtracking` technique to solve the problem.

The main idea is to explore all possible combinations of the given candidates and check if they sum up to the target.

1. We will start by sorting the `candidates` array to handle duplicates easily.

2. We will use a recursive function `backtrack` that takes the current index, the remaining target, and the current combination as parameters.

3. In the `backtrack` function, we will check if the remaining target is less than 0. If it is, we will return immediately as it means we have exceeded the target.

4. If the remaining target is `exactly 0`, it means we have found a `valid` combination. We will add this combination to our result list.

5. We will then iterate through the `candidates` starting from the current index. For each candidate, we will check if it is the same as the previous candidate (to avoid duplicates). If it is, we will skip it.

6. If the candidate is greater than the remaining target, we can break the loop as all subsequent candidates will also be greater (since the array is sorted).

7. If the candidate is valid, we will add it to the current combination and call the `backtrack` function recursively with the next index and the updated remaining target.

8. After the recursive call, we will remove the last added candidate from the current combination to backtrack and explore other possibilities.

---

## Code Implementation

```java
class Solution {
    int n;
    List<List<Integer>> res;
    
    private void backtrack(int index, int[] nums, int target, List<Integer> curr){
        if(target < 0) return;
        if(target == 0){
            res.add(new ArrayList<>(curr));
            return;
        }

        for(int i = index; i < n; i++){
            if(i > index && nums[i - 1] == nums[i]) continue;
            if((target - nums[i]) < 0) return;
            curr.add(nums[i]);
            backtrack(i + 1, nums, target - nums[i], curr);
            curr.remove(curr.size() - 1);
        }
    }

    public List<List<Integer>> combinationSum2(int[] candidates, int target) {
        this.n = candidates.length;
        this.res = new ArrayList<>();
        Arrays.sort(candidates);
        List<Integer> curr = new ArrayList<>();
        backtrack(0, candidates, target, curr);
        return this.res;
    }
}
```

---

## Complexity Analysis

- **Time Complexity**: O(2^n), where `n` is the number of candidates. In the worst case, we might have to explore all possible combinations of candidates.

- **Space Complexity**: O(n), where `n` is the number of candidates. This is because in the worst case, we might have to store a combination of all candidates in the recursion stack.
